# ADR-006: V2 Utility Features Implementation

## Status
Accepted

## Context
Based on ADR-003, users need enhanced utility features for professional use cases: roster reusability, retroactive edits, offline capability, and reporting.

## Decision
Implement comprehensive V2 utility features:

### Roster Management
- Global player name persistence across games
- Quick player selection from history
- Player statistics tracking

### Retroactive Editing
- Enhanced event mutation system
- In-place event editing
- Score correction capabilities

### Offline Mode
- Complete PWA implementation
- Service worker for offline functionality
- Local data synchronization

### Advanced Export
- Multiple export formats (CSV, JSON, PDF)
- Customizable report templates
- Season aggregation capabilities

## Consequences
- ✅ Professional-grade utility features
- ✅ Offline-first architecture
- ✅ Enhanced data management
- ⚠️ Increased local storage usage
- ⚠️ More complex UI navigation

## Implementation
Priority: MEDIUM
Effort: 20 hours
Dependencies: Performance optimization complete
